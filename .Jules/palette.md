## 2024-05-18 - Improve Sidebar accessibility and keyboard navigation
**Learning:** Interactive UI elements built with generic `div` tags lack native keyboard support (tab focus, enter/space interaction) and screen reader identification. Additionally, navigation items that visually hide their text content when collapsed (e.g., in a sidebar) need explicit `aria-label` attributes for screen readers to announce them correctly, along with `aria-current="page"` to indicate active states.
**Action:** Always use semantic interactive elements like `<button>` instead of `<div>` with `onClick` handlers. For elements that conditionally hide text content visually, provide `aria-label`s based on the underlying state to maintain robust screen reader support. Include appropriate focus styles (e.g., `focus-visible:ring-2`) to ensure keyboard navigators can identify the focused element.
